# Internationalization (i18n)

The web app and the user-facing README are localized via [`use-intl`](https://next-intl.dev/docs/environments/core-library) (next-intl's framework-agnostic core). This doc is the single source of truth for: how strings live, what's translated vs. left in English, and how to add a new language.

## Quick map

| Layer | Source of truth | Notes |
|---|---|---|
| UI strings | `apps/web/messages/{locale}.json` | English bundles eagerly; other locales lazy-load as chunks (`apps/web/src/i18n/provider.tsx`) |
| Locale list + names | `apps/web/src/i18n/locales.ts` | Also does `navigator.languages` best-match detection |
| Locale persistence | `localStorage('locale')` | Client-side setting; URLs carry **no** locale prefix |
| Locale-aware navigation | `apps/web/src/i18n/navigation.ts` | `href`-based `Link`/`useRouter` compat over TanStack Router |
| Date / time / number formatting | `useFormatter()` | Replaces dayjs for visible output |
| User-facing README | `README.md` (en, root) + `docs/i18n/README.{locale}.md` (all others) | One language strip at the top of each |
| README translation staleness | `docs/i18n/.translation-status.json` | Records the source hash + each locale's last-synced hash; checked by CI |

## Translation policy

What we translate:
- All UI strings: navigation, page headers, empty states, error/success toasts, form labels and validation messages, button text, breadcrumbs, ARIA labels.
- The README (one file per locale).

What we deliberately *don't* translate:
- **Developer-facing docs** (`CLAUDE.md`, `AGENTS.md`, `CONTRIBUTING.md`, `ARCHITECTURE.md`, `docs/FEATURES.md`, `docs/PROJECT-STRUCTURE.md`, `docs/TESTING.md`, `CHANGELOG.md`). OSS norm; doubling them on every doc PR isn't worth the maintenance.
- **MediaMTX-specific field labels** in the MediaMTX config form (`apps/web/src/features/mediamtx-config/sections/*`). These mirror MediaMTX's own YAML keys (e.g. "RTSP Address", "HLS Server Cert"). Translating them breaks the link from the UI to MediaMTX's documentation. Section titles, tab labels, the server-enable helper descriptions (`sections.*.enableDescription`), save bar, toasts, and buttons *are* translated.
- **PWA manifest name/short_name** (`apps/web/public/manifest.webmanifest`). The product name "MediaMTX Connect" is identical across locales.
- **Brand and proper nouns**: "Connect", "MediaMTX", "RTSP", "RTMP", "HLS", "WebRTC", "SRT", "PPROF".
- **Log messages** (Pino logger output). Logs are operator-facing and stay English.

## Currently supported locales

30 locales, all mirroring `en.json` at 100% key parity (CI-enforced): `en` (source of truth), `es`, `zh` (Simplified), `it`, `de`, `ru`, `fr`, `pt`, `ja`, `pl`, `ko`, `tr`, `nl`, `cs`, `zh-tw` (Traditional), `pt-br`, `id`, `ro`, `sv`, `da`, `no`, `fi`, `el`, `hu`, `uk`, `vi`, `tl`, `th`, `hi`, `bn`.

The locale is a client-side setting: initialized from `navigator.languages`, switched via the sidebar combobox, persisted in `localStorage`. There are no locale-prefixed URLs.

## Adding a new language

1. Add the locale to `locales` and `LANGUAGE_NAMES` in `apps/web/src/i18n/locales.ts`.
2. Copy the source file: `cp apps/web/messages/en.json apps/web/messages/xx.json`.
3. Translate every value in `messages/xx.json`. Do not delete or rename keys — `pnpm i18n:check` enforces parity with `messages/en.json`.
4. Update the README:
   - Create `docs/i18n/README.xx.md` by translating `README.md`. Use `../../` for repo-root references (`ARCHITECTURE.md`, `mediamtx.yml`, `.github/assets/demo.gif`) and `./README.{other}.md` for sibling locales; English links to `../../README.md`.
   - Add a `🇽🇽 <a href="./docs/i18n/README.xx.md">Native</a>` entry to the language strip in `README.md` (root) and a `🇽🇽 <a href="./README.xx.md">Native</a>` entry in every `docs/i18n/README.*.md`.
5. Add an `"xx": "<sourceHash>"` entry to `docs/i18n/.translation-status.json` matching the current `sourceHash`.
6. Update `docs/FEATURES.md` (the §9 Internationalization bullet) and the locale count in this file.
7. Run `pnpm i18n:check` and `pnpm typecheck`. CI will reject the PR if either fails.

### When the source `README.md` changes

`README.md` changes → `sourceHash` shifts → `pnpm i18n:check` fails. Either retranslate the affected locale READMEs and bump their hashes to the new `sourceHash`, or update `sourceHash` alone if the change was English-copy-only and translations have been updated separately. Hash is the first 16 hex chars of `sha256(README.md)`; the script computes and prints the expected value.

## Conventions

- **Namespaces match the surface area**: `Nav`, `Common`, `Streams`, `Recordings`, `Config`, `Forms`, `Metadata`. Add a new namespace when adding a new feature folder.
- **Key style**: dot-nested, camelCase. `Streams.empty.noStreamsTitle`, not `streams_empty_no_streams_title`.
- **ICU placeholders, not concatenation**: `"onlineSince": "Online since {date}"`. Plurals use the ICU `{count, plural, one {...} other {...}}` syntax.
- **Rich text via `t.rich`**: when an element needs to be embedded in copy, the message uses placeholder tags and the call site supplies a renderer:
  ```json
  "noRecordingsHint": "Make sure <code>MTX_RECORD=yes</code> is set in your MediaMTX configuration."
  ```
  ```tsx
  t.rich('noRecordingsHint', { code: chunks => <code>{chunks}</code> })
  ```
- **No raw string literals in JSX** under `apps/web/src/features/**` and `apps/web/src/components/**` (excluding `ui/` shadcn primitives — lint-enforced).
- **`<html lang>` follows the active locale** — set by `I18nProvider` on switch.
- **Date formatting** goes through `useFormatter()`. Don't reach for `dayjs` for visible output (it's still used for date *math* in `apps/web/src/features/recordings/stream-recordings-page.tsx`).

## CI guard

`pnpm i18n:check` runs two scripts in CI:

- `scripts/i18n-check.mjs` — compares every key in `apps/web/messages/{non-en}.json` against `en.json` and exits non-zero if any are missing or extra.
- `scripts/readme-i18n-check.mjs` — hashes `README.md`, compares against `docs/i18n/.translation-status.json`, and exits non-zero if `sourceHash` or any locale's recorded hash is stale, or if a locale README file is missing.

## Translation contract

When you add or change a user-visible string:

1. Add the key to `apps/web/messages/en.json` (the source of truth).
2. Replicate it in **every** non-English locale file.
3. If you don't speak the target language, copy the English value verbatim and mark it with a `TODO:` comment in the JSON file or a tracking issue. `i18n:check` will not catch *untranslated* strings (only *missing* keys), so be honest with yourself.

The CLAUDE.md "hard rule" still applies: if a string change adds, removes, or modifies a user-visible feature, update `docs/FEATURES.md` in the same change.
